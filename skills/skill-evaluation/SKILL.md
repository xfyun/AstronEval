# Skill 评测技能

协助用户对本地 Skill 进行自动化评测，支持单 Skill 验证、多 Skill 横评、多模型横评、运行框架横评等场景。

核心原则：**实现细节对用户不可见**——当不确定是否该展示某信息时，不展示。

不适用场景（引导用户改用其他方式）：代码评审、系统性能压测、模型训练/微调。

---

## 前置规则

执行本 Skill 前，必须加载并遵守 [输出行为规范.md](./references/输出行为规范.md)。加载完毕前禁止向用户输出任何文字。

摘要（完整规则见上述文档）：

1. **首动作 = TodoWrite**，此前禁止任何文字输出
2. **只允许输出**：选项表 / 单行提问 / 确认问句 / 登录提示 / 结果摘要
3. **全程中文**，禁止暴露内部实现（脚本命令、JSON 字段、内部 ID）
4. **静默执行**：鉴权/搜索/打包/上传/轮询等操作全程无输出
5. **禁用 AskUserQuestion**，交互通过文本 + Markdown 表格完成

> **判断标准**：写每一句输出前先自问——这句话是在直接向用户提问、请用户选择、或展示最终结果吗？不是 → 不输出。

**术语约束**（这些词对用户不可见）：

| 用户可见 | 禁止暴露 |
|----------|----------|
| 云端模型、自定义模型 | `TokenPlan`、`EvalPlan`、`limited_free` |
| 驱动模型、评委模型 | `candidate`、`judge`、`usage`、`source` |

## 处理阶段

| 阶段 | 文档 | 核心任务 |
|------|------|----------|
| 初始化 | [eval-init.md](./eval-init.md) | 鉴权验证、历史任务加载、意图分析、场景确认 |
| 构建配置 | [eval-build.md](./eval-build.md) | 选择 Skill、运行框架、驱动模型、评委模型 |
| 数据准备 | [eval-set.md](./eval-set.md) | 上传或自动合成评测集、预览确认 |
| 执行评测 | [eval-execute.md](./eval-execute.md) | 任务摘要、提交、轮询、展示报告 |

- 默认按序推进：阶段1 → 阶段2 → 阶段3 → 阶段4
- 用户明确要求时可跳转到指定阶段
- 阶段内任务全部完成后自动进入下一阶段
- 每进入新阶段/新步骤，调用 TodoWrite 更新进度（格式见 [进度展示规范.md](./references/进度展示规范.md)）

## 入口行为

Skill 加载后的首轮响应：

1. 调用 TodoWrite 更新阶段1进度（格式见 [进度展示规范.md](./references/进度展示规范.md)）
2. 静默执行鉴权检查
3. 根据条件决定首个用户可见输出：

| 条件 | 首个输出 |
|------|----------|
| Token 失效需手动授权 | 登录链接 |
| 用户已描述具体需求 → 新建评测 | 场景推断确认（"根据你的描述，你想要「xxx」，是这样吗？"） |
| 用户未描述具体需求 + 有历史任务 | 历史任务表 |
| 用户未描述具体需求 + 无历史任务 | 自然语言引导（"请描述你的评测需求，例如：…"） |

**硬性约束**：阶段1只问场景，不问模型/评测集等阶段2/3的问题。
