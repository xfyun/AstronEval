# 进度展示规范

本文档定义 Skill 评测过程中向用户展示进度的格式与规范。

---

## 核心原则

| 原则 | 说明 |
|------|------|
| **渐进式披露** | 默认输出简化信息，完整信息按需展示 |
| **情境感知** | 根据用户行为动态调整输出详细程度 |
| **路径可回溯** | 展示"已走过的路径"帮助用户理解上下文 |

---

## 展示层级

| 时机 | 展示内容 |
|------|----------|
| 阶段1（场景确认） | **不展示进度表**，先做场景确认，避免打扰用户首次决策 |
| 场景确认后、进入阶段2 之前 | 调用 TodoWrite 打印流程入口表（# 序号格式），标志阶段1 完成、阶段2 开始 |
| 进入阶段3、4 | 调用 TodoWrite 更新 4 阶段表（completed/in_progress/pending 状态）|
| 进入新步骤 | 不展示进度 |

---

## 表格渲染格式

**进度表使用 TodoWrite 工具渲染**，会自动展示为带状态的任务列表，已完成任务会显示删除线。

### TodoWrite 调用示例

**阶段1入口**：

```
TodoWrite([
  { content: "确认评测场景", status: "in_progress", activeForm: "检查登录状态" },
  { content: "配置评测对象", status: "pending", activeForm: "配置评测对象" },
  { content: "准备评测数据", status: "pending", activeForm: "准备评测数据" },
  { content: "提交并查看报告", status: "pending", activeForm: "提交并查看报告" }
])
```

**阶段2 步骤执行中**：

```
TodoWrite([
  { content: "确认评测场景", status: "completed", activeForm: "检查登录状态" },
  { content: "配置评测对象", status: "in_progress", activeForm: "上传 Skill 包" },
  { content: "准备评测数据", status: "pending", activeForm: "准备评测数据" },
  { content: "提交并查看报告", status: "pending", activeForm: "提交并查看报告" }
])
```

**阶段4 步骤执行中**：

```
TodoWrite([
  { content: "确认评测场景", status: "completed", activeForm: "检查登录状态" },
  { content: "配置评测对象", status: "completed", activeForm: "确认评委模型" },
  { content: "准备评测数据", status: "completed", activeForm: "确认评测数据" },
  { content: "提交并查看报告", status: "in_progress", activeForm: "等待评测完成" }
])
```

> ⚠️ 状态说明：`completed` = 已完成（带删除线）、`in_progress` = 执行中、`pending` = 待执行。
> `activeForm` 为用户友好的步骤描述，避免暴露技术细节（如"检查 Token"、"查询数据库"等）。

### 各步骤 activeForm 对照表

| 阶段 | 步骤 | activeForm |
|------|------|------------|
| 阶段1 | 检查 Token | 检查登录状态 |
| 阶段1 | 触发登录 | 等待浏览器授权 |
| 阶段1 | 查询历史任务 | 查询历史任务 |
| 阶段1 | 识别评测场景 | 分析评测需求 |
| 阶段2 | 选择 Skill 获取方式 | 确认 Skill 来源 |
| 阶段2 | 搜索/识别 Skill | 确认 Skill 中 |
| 阶段2 | 打包上传 Skill | 上传 Skill 包 |
| 阶段2 | 选择运行框架 | 选择运行框架 |
| 阶段2 | 选择驱动模型 | 选择驱动模型 |
| 阶段2 | 平台推荐评委模型 | 确认评委模型 |
| 阶段2 | 选择其他评委模型 | 选择评委模型 |
| 阶段3 | 选择评测集来源 | 准备评测数据 |
| 阶段3 | 上传评测集 | 上传评测数据 |
| 阶段3 | 合成评测集 | 生成评测数据 |
| 阶段3 | 预览评测集 | 确认评测数据 |
| 阶段4 | 展示任务摘要 | 确认评测配置 |
| 阶段4 | 提交评测任务 | 提交评测任务 |
| 阶段4 | 等待评测结果 | 等待评测完成 |
| 阶段4 | 展示评测报告 | 查看评测报告 |

**更新时机**：每进入一个新步骤，立即调用 TodoWrite 更新当前阶段的 activeForm。

**原则**：activeForm 使用用户友好的描述，避免暴露技术细节（如"检查 Token"、"查询数据库"、"调用 API"等）。

**禁止**：content 字段固定为 4 个阶段名称，不得使用用户意图或自由文本（如"评测 xxx 效果"）。

### 其他表格格式（候选项、任务摘要等）

**候选项、任务摘要、结果摘要、历史任务列表等表格统一使用 Markdown pipe 表格格式**（`| col | col |`），由渲染环境自动对齐。

## 流程入口表

打印时机分两种：

| 场景类型 | 时机 |
|---------|------|
| 场景明确（推断确认）| 先做场景推断确认（二选一）→ 用户确认 → **再调用 TodoWrite 打印流程入口表** → 进入阶段2 |
| 场景模糊（5 选 1）| 调用 TodoWrite 打印流程入口表 → **紧接着同一轮**展示 5 场景选择表 → 用户选 → 进入阶段2 |

**TodoWrite 调用格式**（阶段2入口）：

```
TodoWrite([
  { content: "确认评测场景", status: "completed", activeForm: "检查登录状态" },
  { content: "配置评测对象", status: "in_progress", activeForm: "确认 Skill 来源" },
  { content: "准备评测数据", status: "pending", activeForm: "准备评测数据" },
  { content: "提交并查看报告", status: "pending", activeForm: "提交并查看报告" }
])
```

场景确定后立即进入阶段 2（配置评测对象）。

---

## 进入新阶段时

**仅在进入阶段3、4 时**按下方顺序输出（阶段1 不打印进度表；阶段2 由流程入口表覆盖，也不重复打印）：

| 顺序 | 操作 | 说明 |
|------|------|------|
| 1 | 调用 TodoWrite 更新进度表 | 使用 completed/in_progress/pending 状态标记各阶段 |
| 2 | 执行具体步骤 | 调用脚本、读取文件等 |

**TodoWrite 调用格式**（阶段3入口）：

```
TodoWrite([
  { content: "确认评测场景", status: "completed", activeForm: "检查登录状态" },
  { content: "配置评测对象", status: "completed", activeForm: "确认评委模型" },
  { content: "准备评测数据", status: "in_progress", activeForm: "准备评测数据" },
  { content: "提交并查看报告", status: "pending", activeForm: "提交并查看报告" }
])
```

## 任务摘要格式（阶段4 步骤1）

```
📍 评测任务摘要

| 配置项 | 内容 |
|--------|------|
| Skill 名称 | {skill-name} |
| 运行框架 | {runtime-type-1}、{runtime-type-2} |
| 驱动模型 | {model-display-name-1}、{model-display-name-2} |
| 评测集 | {n} 条用例，具体见 {local-dataset-path} |
| 评委模型 | {judge-display-name} |
| 预估耗时 | 约 {estimated-minutes} 分钟 |

确认以上配置，提交评测任务？
```

**预估耗时计算**：每条用例约 1.5~2 分钟，多个驱动模型时并行执行但总耗时仍与模型数正相关。
公式：`ceil(n * 1.8 * m)` 分钟，其中 `n` 为用例数，`m` 为驱动模型数量，向上取整。

## 历史任务列表格式（阶段1 任务2）

```
📋 最近的评测任务

| # | 状态 | Skill 名称 | 驱动模型 | Task ID | 执行进度 | 提交时间 |
|:----:|------|-----------|----------|---------|---------|----------|
| 1 | 已完成 | {skill-name} | {model} | {task_id} | {progress}% | 2026-05-12 14:23:08 |
| 2 | 执行中 | {skill-name} | {model} | {task_id} | {progress}% | 2026-05-11 09:45:31 |
| 3 | 配置中 | {skill-name} | {model} | - | - | 2026-05-10 18:02:55 |

输入序号{动态文案}，或输入"新建"开始新的评测。
```

> 提交时间统一展示为 `YYYY-MM-DD HH:MM:SS`（精确到秒），数据源自 `tasks.json` 的 `submitted_at`。

---

## 候选项格式（多处复用）

所有需要用户选择的场景统一使用编号表格：

```
| 选项 | {分类列}... | 说明 |
|:----:|-------------|------|
| 1 | xxx | ... |
| 2 | yyy | ... |
```

询问语句末尾要明确：

| 场景 | 询问句式 |
|------|----------|
| 单选 | `请选择...（输入序号）：` |
| 多选 | `请选择...（输入序号，可多选，逗号分隔，至少 N 个）：` |
| 二选一确认 | 不带"输入"二字，直接 `请选择：` 后附两选项表 |

## 静默执行清单

完整规则见 [输出行为规范.md](./输出行为规范.md)。以下为各阶段静默动作速查：

| 动作 | 所在阶段 |
|------|----------|
| Token 检查与登录 | 阶段1 任务1 |
| Skill 搜索、下载、依赖识别、打包、上传 | 阶段2 任务1 |
| 运行框架配置构建 | 阶段2 任务2 |
| 模型列表拉取、自定义模型保存 | 阶段2 任务3、任务4 |
| Excel 转换、评测集上传、删除/编辑同步 | 阶段3 任务1、任务2 |
| 平台数据合成轮询 | 阶段3 任务1 |
| 评测任务提交、状态轮询、artifacts 拉取 | 阶段4 任务1～3 |

仅在完成节点输出最终结果。
