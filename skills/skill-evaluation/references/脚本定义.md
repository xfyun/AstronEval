# 脚本定义

本文档定义 `eval_skill.py` 各子命令的参数说明与使用示例。

> **执行约定**：所有命令均通过 `{python-cmd} "{skill-dir}/scripts/eval_skill.py" <subcommand>` 调用。
> Windows 环境下，所有路径参数必须用双引号包裹。
>
> **统一鉴权**：所有调用评测平台 API 的子命令（`models`/`package`/`upload*`/`upload-dataset`/`datamaker*`/`submit`/`status`/`artifacts`/`list-dataset`/`edit-dataset`/`delete-cases`/`convert-excel`）都会自动从 `--auth-file`（默认 `./.eval/auth.json`）读取 Token 并注入 `Authorization: Bearer` 头。Token 缺失或过期时直接返回错误，需先通过 `login` 子命令完成授权。如评测会话与默认路径不一致，可显式传 `--auth-file "{work-dir}/.eval/auth.json"`。
>
> **鉴权平台**：使用讯飞开放平台鉴权（`base_url: https://maas.xfyun.cn`），`client_id` 已内置在脚本中。

| 历史任务 | `recent-tasks` | 查询最近任务 |
| 评测集 | `convert-excel`, `upload-dataset`, `datamaker`, `datamaker-status`, `list-dataset`, `delete-cases` | 评测集生命周期 |
| 任务管理 | `submit`, `status`, `summary`, `artifacts` | 提交、轮询、摘要、报告 |

## 1. 鉴权与登录

### check-token

```bash
{python-cmd} "{skill-dir}/scripts/eval_skill.py" check-token --auth-file "{work-dir}/.eval/auth.json"
```

输出 `status`：`valid` / `invalid` / `not_found`。

### check-dataset

检查评测集字段完整性：

```bash
{python-cmd} "{skill-dir}/scripts/eval_skill.py" check-dataset \
  --dataset "{work-dir}/.eval/{session-id}/eval-dataset.json"
```

| 输出字段 | 说明 |
|----------|------|
| `needs_completion` | `true` 表示需要补全（缺少字段或存在空值） |
| `missing_fields` | 缺少的字段列表（`category` / `keypoint`） |
| `empty_category_count` | category 为空的样例数量 |
| `empty_keypoint_count` | keypoint 为空的样例数量 |

### login

```bash
{python-cmd} "{skill-dir}/scripts/eval_skill.py" login --auth-file "{work-dir}/.eval/auth.json"
```

输出 `status`：`success`（自动回调） / `manual_url`（需用户手动输入授权码）。

当返回 `manual_url` 时，会同时返回 `login_url` 和 `state_token`。用户在浏览器完成授权后输入授权码，调用此命令完成登录。

### token

```bash
{python-cmd} "{skill-dir}/scripts/eval_skill.py" token \
  --auth-file "{work-dir}/.eval/auth.json" \
  --code "{user-input-code}" \
  --state-token "{state-token}"
```

| 参数 | 说明 |
|------|------|
| `--code` | 用户从浏览器获取的授权码 |
| `--state-token` | login 返回的 state_token |

输出 `status`：`success`（Token 已保存） / `error`（授权码无效）。

## 2. Skill 管理

### search-skill

> ⚠️ **参数注意**：必须用 `--name "{skill-name}"` 格式，不能直接传 skill 名称作为位置参数。

```bash
{python-cmd} "{skill-dir}/scripts/eval_skill.py" search-skill --name "{skill-name}" --work-dir "{work-dir}"
```

返回本地 + 云端搜索结果（JSON）。**路径字段需经过 JSON 解析后再展示**，不要做字符串二次处理。

**本地搜索匹配策略**（渐进式降级）：

| 优先级 | match_type | 说明 |
|--------|------------|------|
| 1 | `exact` | 目录名与 `--name` 完全一致 |
| 2 | `substring` | 目录名包含 `--name`（大小写不敏感） |
| 3 | `fuzzy` | 相似度 ≥ 0.6（附带 `score` 字段） |

高优先级有结果时不返回低优先级。

### download-skill

> ⚠️ **参数注意**：使用 `--slug`（非 `--name`），必须传 `--output-dir`。

```bash
{python-cmd} "{skill-dir}/scripts/eval_skill.py" download-skill --slug "{skill-name}" --output-dir "{work-dir}/.eval/{session-id}"
```

### package（按场景分三种调用方式）

> ⚠️ **参数注意**：使用 `--skill-dir`（非 `--skill-path`），必须传 `--config` 和 `--output-dir`。

**单目录打包上传**：
```bash
{python-cmd} "{skill-dir}/scripts/eval_skill.py" package \
  --config "{skill-dir}/scripts/cfg/eval-server.cfg" \
  --skill-dir "{target-skill-dir}" \
  --output-dir "{work-dir}/.eval/{session-id}"
```

**父 + 协同/依赖（合成单 zip）**：
```bash
{python-cmd} "{skill-dir}/scripts/eval_skill.py" package \
  --config "{skill-dir}/scripts/cfg/eval-server.cfg" \
  --skill-dir "{parent-skill-dir}" \
  --extra-skills "{dep-dir-1},{dep-dir-2}" \
  --output-dir "{work-dir}/.eval/{session-id}"
```

**直接上传已有 zip**（云端下载或用户提供）：
```bash
{python-cmd} "{skill-dir}/scripts/eval_skill.py" package \
  --config "{skill-dir}/scripts/cfg/eval-server.cfg" \
  --skill-zip "{skill-zip-path}" \
  --output-dir "{work-dir}/.eval/{session-id}"
```

输出 `{skill-zip-url}`。

## 3. 会话管理

### session

> ⚠️ **参数注意**：使用 `--action create`（非 `--create`），必须传 `--work-dir`。

```bash
{python-cmd} "{skill-dir}/scripts/eval_skill.py" session \
  --work-dir "{work-dir}" \
  --action create \
  --skill-name "{skill-name}"
```

会话目录命名规则：`YYYYMMDD_HHMMSS_{skill-name}`。

## 4. 模型管理

### models

拉取 TokenPlan / EvalPlan 模型列表。**必须传 `--config`**：

```bash
# 驱动模型候选（TokenPlan）
{python-cmd} "{skill-dir}/scripts/eval_skill.py" models \
  --config "{skill-dir}/scripts/cfg/eval-server.cfg" \
  --scene skill --usage candidate

# 评委模型候选（EvalPlan）
{python-cmd} "{skill-dir}/scripts/eval_skill.py" models \
  --config "{skill-dir}/scripts/cfg/eval-server.cfg" \
  --scene skill --usage judge
```

### custom-models

```bash
# 读取历史
{python-cmd} "{skill-dir}/scripts/eval_skill.py" custom-models \
  --work-dir "{work-dir}" \
  --action read --usage {candidate|judge}

# 添加新模型
{python-cmd} "{skill-dir}/scripts/eval_skill.py" custom-models \
  --work-dir "{work-dir}" \
  --action add --usage {candidate|judge} \
  --model-id {id} --api-key {key} --api-url {url}
```

### build-runtimes

构建运行框架配置（生成 `eval-runtimes.json`）：

```bash
{python-cmd} "{skill-dir}/scripts/eval_skill.py" build-runtimes \
  --output-dir "{work-dir}/.eval/{session-id}" \
  --runtimes {runtime-type-1},{runtime-type-2}
```

| 参数 | 说明 |
|------|------|
| `--runtimes` | 运行框架类型，多个以逗号分隔（默认 `claude-code`）；支持：`claude-code`、`openclaw`、`hermes` |

### build-models

> ⚠️ **参数注意**：使用 `--output-dir`（非 `--output`），`--judge-json` 和 `--models-json` 是 JSON 字符串，需用单引号包裹。

构建模型配置（生成 `eval-models.json` 与 `eval-judge.json`）：

```bash
{python-cmd} "{skill-dir}/scripts/eval_skill.py" build-models \
  --output-dir "{work-dir}/.eval/{session-id}" \
  --judge-json '{judge-model-object-json}' \
  --models-json '{driver-models-array-json}' \
  --work-dir "{work-dir}" \
  --session-id "{session-id}"
```

| 参数 | 说明 |
|------|------|
| `--judge-json` | 阶段2 任务4 选定的评委模型对象（单个 JSON） |
| `--models-json` | 阶段2 任务3 选定的驱动模型对象数组（JSON 数组） |
| `--work-dir` | 工作目录，用于刷新 `tasks.json` 中本会话的 `models` 字段（可选） |
| `--session-id` | 会话 ID；缺省时按 `--output-dir` 推断（可选） |

```bash
{python-cmd} "{skill-dir}/scripts/eval_skill.py" recent-tasks \
```

## 6. 评测集管理

### convert-excel

> ⚠️ **参数注意**：使用 `--file`（非 `--excel` 或 `--input`），必须传 `--output` 和 `--config`。

```bash
{python-cmd} "{skill-dir}/scripts/eval_skill.py" convert-excel \
  --file "{excel-path}" \
  --output "{work-dir}/.eval/{session-id}/eval-dataset.json" \
  --config "{skill-dir}/scripts/cfg/eval-server.cfg"
```

### upload-dataset

> ⚠️ **参数注意**：使用 `--dataset`（非 `--file` 或 `--input`），必须传 `--config` 和 `--output`。

```bash
# 首次上传
{python-cmd} "{skill-dir}/scripts/eval_skill.py" upload-dataset \
  --config "{skill-dir}/scripts/cfg/eval-server.cfg" \
  --dataset "{work-dir}/.eval/{session-id}/eval-dataset.json" \
  --output "{work-dir}/.eval/{session-id}/eval-dataset-meta.json"

# 整份覆盖（编辑后）
{python-cmd} "{skill-dir}/scripts/eval_skill.py" upload-dataset \
  --config "{skill-dir}/scripts/cfg/eval-server.cfg" \
  --dataset "{work-dir}/.eval/{session-id}/eval-dataset.json" \
  --dataset-id {dataset-id} \
  --output "{work-dir}/.eval/{session-id}/eval-dataset-meta.json"
```

### datamaker

```bash
{python-cmd} "{skill-dir}/scripts/eval_skill.py" datamaker \
  --config "{skill-dir}/scripts/cfg/eval-server.cfg" \
  --skill-names {skill-name-1},{skill-name-2} \
  --skill-urls {skill-zip-url-1},{skill-zip-url-2} \
  --output "{work-dir}/.eval/{session-id}/eval-datamaker.json"
```

| 参数 | 是否必填 | 说明 |
|------|----------|------|
| `--skill-names` | 是 | skill 名称，多个以逗号分隔；顺序需与 `--skill-urls` 一一对齐 |
| `--skill-urls` | 是 | skill zip 包 URL，多个以逗号分隔；顺序需与 `--skill-names` 一一对齐 |
| `--count` | 否 | 期望合成的样例数（正整数）；不传则由平台自动决定 |

### datamaker-status

```bash
{python-cmd} "{skill-dir}/scripts/eval_skill.py" datamaker-status \
  --config "{skill-dir}/scripts/cfg/eval-server.cfg" \
  --id {datamaker-id} \
```

### list-dataset

```bash
{python-cmd} "{skill-dir}/scripts/eval_skill.py" list-dataset \

  --dataset-id {dataset-id} \
```

```bash

  --config "{skill-dir}/scripts/cfg/eval-server.cfg" \
  --dataset-id {dataset-id} \
```

## 7. 任务管理

### submit

> ⚠️ **参数注意**：`--skill-names` 和 `--skill-urls` 用逗号分隔多个值（非空格），必须传 `--config`、`--dataset-id`、`--models`、`--output`。

```bash
{python-cmd} "{skill-dir}/scripts/eval_skill.py" submit \
  --config "{skill-dir}/scripts/cfg/eval-server.cfg" \
  --skill-names {skill-name-1},{skill-name-2} \
  --skill-urls {skill-zip-url-1},{skill-zip-url-2} \
  --dataset-id {dataset-id} \
  --models "{work-dir}/.eval/{session-id}/eval-models.json" \
  --judge "{work-dir}/.eval/{session-id}/eval-judge.json" \
  --runtime {runtime-type-1},{runtime-type-2} \
  --output "{work-dir}/.eval/{session-id}/eval-task.json" \
  --work-dir "{work-dir}"
```

| 参数 | 是否必填 | 说明 |
|------|----------|------|
| `--skill-names` | 是 | 待评测 skill 名称，多个以逗号分隔；顺序需与 `--skill-urls` 一一对齐 |
| `--skill-urls` | 否 | skill zip 包 URL，多个以逗号分隔；若提供，数量必须与 `--skill-names` 相同 |
| `--runtime` | 否 | 运行框架类型，多个以逗号分隔（默认 `claude-code`）；支持：`claude-code`、`openclaw`、`hermes` |

### status（后台轮询）

> ⚠️ **参数注意**：必须传 `--config`、`--task-id`、`--output`。轮询模式需加 `--poll`。

使用 Bash 工具，`run_in_background: true`，`timeout: 600000`：

```bash
{python-cmd} "{skill-dir}/scripts/eval_skill.py" status \
  --config "{skill-dir}/scripts/cfg/eval-server.cfg" \
  --task-id {task-id} \
  --output "{work-dir}/.eval/{session-id}/eval-result.json" \
  --poll --interval 30 --timeout 3600 \
  --work-dir "{work-dir}"
```

### summary

```bash
{python-cmd} "{skill-dir}/scripts/eval_skill.py" summary \
  --result "{work-dir}/.eval/{session-id}/eval-result.json"
```

自动从 `artifacts` 中提取 `report_render_url` 作为在线报告链接。

### artifacts（按需）

若 `eval-result.json` 中 `artifacts` 为空：

```bash
{python-cmd} "{skill-dir}/scripts/eval_skill.py" artifacts \
  --config "{skill-dir}/scripts/cfg/eval-server.cfg" \
  --task-id {task-id}
```

## 失败处理通则

| 退出码 | 含义 | 处理 |
|--------|------|------|
| 0 | 成功 | - |
| 非 0 | 失败 | 读取脚本 stderr，将错误信息提取并按"失败处理"展示给用户 |

具体错误码与含义见服务端约定。
