# 初始化阶段


## 静默鉴权

**后台静默执行 Token 检查**，不输出任何提示。

---

## ⚠️ 首个输出规则

**打印进度表并完成鉴权后，根据触发条件展示对应的首个输出**：

| 条件 | 首个输出 |
|------|----------|
| 有历史任务且用户未描述具体需求 | 历史任务表 |
| 用户从历史任务表选择"新建"（即使本来有历史任务，选了"新建"后视同未描述具体需求）| 自然语言引导（"请描述你的评测需求，例如：…"）|
| 用户描述中**场景信号明确**（如"对比 v1 v2 版本"、"哪个模型驱动效果更好"、"测试 xxx 的效果"） | 场景推断确认（二选一）|
| 用户描述中**场景模糊**（仅给 Skill 名称、仅说"评测"、信号不唯一） | 4 场景选择表|
| 无历史任务且用户未描述具体需求 | 自然语言引导（"请描述你的评测需求，例如：…"）|
| Token 失效需手动授权 | 登录链接（manual_url 场景） |

> ⚠️ **历史任务优先级最高**：只要存在历史任务且**用户本轮未描述具体评测需求**，**首个输出必须是历史任务表**，不能跳过直接打印自然语言引导。即使用户之前的会话有过描述，只要本轮触发时输入是空或泛化（如直接 `/astron-eval`、"看看评测"），都必须先展示历史任务表，让用户选"继续某任务"或"新建"。
>
> ⚠️ **"已描述具体需求"的严格定义**：必须包含**至少一项具体信息**才算已描述需求：
>
> | 类型 | 示例 |
> |------|------|
> | 具体的 Skill 名称或路径 | "评测 xxx Skill"、"D:/path/to/skill" |
> | 具体的驱动/评委模型 | "Claude-5 跑 xxx"、"用 DeepSeek-V3" |
> | 明确的场景信号（含场景关键词且能映射到 4 种场景之一）| "对比 v1 v2 版本"、"哪个模型驱动效果更好"、"多个 Skill 横评" |
>
> 仅泛化触发语（如"我想评测"、"评测 skill"、"做个评测"、"想试试评测"）**不算已描述具体需求**——必须按"用户未描述具体需求"分支处理（先展示历史任务表）。

---

## 目标

完成鉴权验证、历史任务加载、意图识别、评测场景确认后，进入构建配置阶段。

核心原则：**静默优先，按序执行，失败即止**。

## 何时使用

- 用户首次触发 skill-evaluation
- 用户希望继续/恢复历史评测任务
- 鉴权状态不明时

---

## 阶段完成标志

**验证顺序**（按序执行，任一失败则执行对应任务）：

1. ✅ Token 有效（`{work-dir}/.eval/auth.json`） → 否则执行任务1
2. ✅ 已识别用户意图（新建 / 继续旧任务） → 否则执行任务2
3. ✅ 评测场景已明确（4 种场景之一） → 否则执行任务3

全部通过后，进入构建配置阶段（加载 [eval-build.md](./eval-build.md)）。

---

## 任务列表

### 任务1：鉴权与登录

#### 步骤1：检查 Token

后台静默执行：

```bash
{python-cmd} "{skill-dir}/scripts/eval_skill.py" check-token --auth-file "{work-dir}/.eval/auth.json"
```

| 输出 status | 说明 | 后续动作 |
|-------------|------|----------|
| `valid` | Token 有效 | 完成本任务 |
| `invalid` / `not_found` | 失效或文件不存在 | 进入步骤2 |

---

#### 步骤2：触发登录

后台静默执行：

```bash
{python-cmd} "{skill-dir}/scripts/eval_skill.py" login --auth-file "{work-dir}/.eval/auth.json"
```

| 输出 status | 说明 | 后续动作 |
|-------------|------|----------|
| `success` | 自动回调成功 | 完成本任务 |
| `manual_url` | 需用户手动复制授权码 | 进入步骤3 |

#### 步骤3：等待用户输入授权码

向用户展示：

> 请访问登录链接完成授权，然后输入授权码：
>
> {login_url}

等待用户输入授权码后，后台静默调用 `token` 子命令完成登录：

```bash
{python-cmd} "{skill-dir}/scripts/eval_skill.py" token \
  --auth-file "{work-dir}/.eval/auth.json" \
  --code "{user-input-code}" \
  --state-token "{state-token}"
```

| 输出 status | 说明 | 后续动作 |
|-------------|------|----------|
| `success` | 授权码已保存到 auth.json | 完成本任务 |
| `error` | 授权码无效或过期 | 提示用户重新访问链接获取新授权码 |

> **⚠️ 不可跳过**：用户必须在浏览器完成授权后输入授权码，无法绕过。

---

### 任务2：识别用户意图

#### 步骤1：判断触发方式

| 触发方式 | 判断依据 | 后续动作 |
|----------|----------|----------|
| 用户已描述具体需求 | 提及 Skill 名称、模型名称或评测场景 | 跳过历史任务展示 → 进入任务3 |
| 用户未描述具体需求 | 手动触发或仅说"看看评测"等 | 加载历史任务 → 步骤2 |

---

#### 步骤2：展示最近任务

后台静默执行：

```bash
{python-cmd} "{skill-dir}/scripts/eval_skill.py" recent-tasks --work-dir "{work-dir}" --limit 5
```

按下方格式展示：

> 📋 最近的评测任务
>
> | # | 状态 | Skill 名称 | 驱动模型 | Task ID | 执行进度 | 提交时间 |
> |:----:|------|-----------|----------|---------|---------|----------|
> | 1 | 已完成 | {skill-name} | {model} | {task_id 或 -} | {progress}% 或 - | 2026-05-12 14:30:25 |
> | 2 | 执行中 | {skill-name} | {model} | {task_id 或 -} | {progress}% 或 - | 2026-05-11 09:15:42 |
> | 3 | 配置中 | {skill-name} | {model} | - | - | 2026-05-10 16:45:00 |
>
> 输入序号{动态文案}，或输入"新建"开始新的评测。

**字段说明**：

| 字段 | 含义 |
|------|------|
| Task ID | 任务提交后由服务端分配；配置中状态显示 `-` |
| 执行进度 | 已完成任务从结果文件读取（如 `100%`）；执行中无缓存显示 `-`；配置中显示 `-` |

**状态枚举**：

| 状态 | 含义 |
|------|------|
| 配置中 | 会话已创建，尚未提交任务 |
| 执行中 | 任务已提交，等待结果 |
| 已完成 | 评测结束（含成功/失败） |

**动态文案规则**：若有已完成任务则提示"查看报告或继续评测"，若有执行中/配置中任务则提示"继续评测"。

| 用户选择 | 后续动作 |
|----------|----------|
| 已完成的任务 | 跳到阶段4 步骤2，展示报告 |
| 执行中的任务 | 跳到阶段4 步骤1 后段，继续轮询 |
| 配置中的任务 | 恢复到对应阶段继续配置 |
| 输入"新建" | 进入任务3 |

若无最近任务，自动进入任务3。

---

### 任务3：识别评测场景

执行 [scene-detection.md](./processes/scene-detection.md) 流程，完成意图分析与场景确认。

#### 子流程出口

| 出口 | 说明 | 后续动作 |
|------|------|----------|
| 场景已明确 | 4 种评测场景之一 | 完成本任务，向用户展示流程概览，进入构建配置阶段 |
| 场景模糊 | 用户选择"不确定" | 在 scene-detection 内继续引导，直至场景明确 |

**4 种评测场景**：

| 场景 | Skill 数量 | 驱动模型数量 |
|------|-----------|-------------|
| 快速效果验证 | 1 个 | 1 个 |
| 同类 Skill 横评 | 多个 | 1 个 |
| 驱动模型横评 | 1 个 | 多个 |
| Skill 版本对比、A/B Test | 多个（同 Skill 不同版本） | 1 个 |

详见 [评测场景说明.md](./references/评测场景说明.md)。

## Red Flags

| 违规行为 | 简洁理由 |
|----------|----------|
| 跳过 Token 检查直接进入业务流程 | 鉴权是前置条件 |
| 用户未描述需求时不展示历史任务 | 历史任务能让用户直接恢复或查看，提升体验 |
| 用户已描述需求时仍展示历史任务 | 增加干扰，违反自动填充原则 |
| 在向用户输出"鉴权检查"等中间提示 | 静默执行原则 |

> 通用违规行为见 [SKILL.md Red Flags](./SKILL.md#red-flags)

---

## 常见错误

| 错误 | 原因 | 解决方案 |
|------|------|----------|
| Token 文件存在但无效 | Token 已过期 | 触发重新登录 |
| 浏览器无法自动打开 | 服务器终端无图形界面 | 使用 manual_url 模式手动复制授权码 |
| 用户多次未回填授权码 | 用户离开 | 提示用户重新触发，或保留任务等待返回 |
