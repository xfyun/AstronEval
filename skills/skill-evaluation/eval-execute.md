# 提交并查看报告阶段

## 目标

完成任务摘要确认、提交、状态轮询、报告展示后，评测流程结束。

核心原则：**摘要必经，提交后台化，报告链接必给**。

**前置验证**：Token 有效；至少 1 个 Skill zip URL 已就绪；`eval-models.json`、`eval-judge.json`、`eval-dataset-meta.json` 都已存在。验证失败则返回对应前置阶段。

---

## 何时使用

- 数据准备阶段已完成（评测集就绪）
- 需要提交、查询或展示评测任务时

---

## 阶段完成标志

**验证顺序**（按序执行，任一失败则执行对应任务）：

1. ✅ `eval-task.json` 存在（含 `task-id`） → 否则执行任务1
2. ✅ 任务状态为 `Succeeded` 或 `Failed` → 否则执行任务2
3. ✅ 评测结果已展示给用户 → 否则执行任务3

全部通过后，评测流程结束。

---

## 任务列表

### 任务1：提交评测任务

**用户视角**：确认任务摘要后提交，无需关心后台操作。

#### 步骤1：展示任务摘要

向用户展示：

> 📍 评测任务摘要
>
> | 配置项 | 内容 |
> |--------|------|
> | Skill 名称 | {skill-name} |
> | 运行框架 | {runtime-type-1}、{runtime-type-2}、... |
> | 驱动模型 | {model-display-name-1}、{model-display-name-2}、... |
> | 评测集 | {n} 条用例，具体见 {local-dataset-path} |
> | 评委模型 | {judge-display-name} |
> | 预估耗时 | 约 {estimated-minutes} 分钟 |
>
> 确认以上配置，提交评测任务？
>
> | 选项 | |
> |:----:|--|
> | 1 | 提交任务 |
> | 2 | 修改配置 |

> ⚠️ **重要**：
> - 运行框架、驱动模型必须展示用户在阶段2选择的**全部**选项（横评场景为多个，单选场景为 1 个），用顿号分隔
> - 评测集行必须包含 `具体见 {local-dataset-path}`，让用户能找到完整数据

| 用户选择 | 后续动作 |
|----------|----------|
| 1 - 提交任务 | 进入步骤2 |
| 2 - 修改配置 | 展示修改选项表 |

用户选择"修改配置"后，展示：

> 请选择要修改的配置项：
>
> | 选项 | 配置项 |
> |:----:|--------|
> | 1 | Skill |
> | 2 | 运行框架 |
> | 3 | 驱动模型 |
> | 4 | 评委模型 |
> | 5 | 评测集 |

| 用户选择 | 后续动作 |
|----------|----------|
| 1 - Skill | 跳回阶段2 任务1 |
| 2 - 运行框架 | 跳回阶段2 任务2 |
| 3 - 驱动模型 | 跳回阶段2 任务3 |
| 4 - 评委模型 | 跳回阶段2 任务4 |
| 5 - 评测集 | 跳回阶段3 任务2 |

**预估耗时计算**：每条用例约 1.5~2 分钟，多个驱动模型或运行框架时并行执行但总耗时与两者数量正相关。
公式：`ceil(n * 1.8 * m * r)` 分钟，其中 `n` 为用例数，`m` 为驱动模型数量，`r` 为运行框架数量，向上取整。

#### 步骤2：提交评测任务

后台静默执行（具体命令见 [脚本定义.md](./references/脚本定义.md#7-任务管理)）：

1. 提交评测任务（生成 `eval-task.json`）
2. 启动后台轮询（Bash 工具，`run_in_background: true`，`timeout: 600000`）

提交成功后，**逐字原样**输出以下一句话，不得添加 task-id、轮询状态等任何额外信息：

> 评测任务已提交，完成后会自动展示报告。

**禁止**输出"评测任务已提交（task-id: xxx）"、"后台轮询中"、"你可以继续做别的事"等变体。

#### 步骤3：等待结果

后台任务在轮询过程中静默更新 `eval-result.json`。本任务完成节点 = 后台轮询任务返回，进入任务2。

---

### 任务2：监控状态

**用户视角**：用户可继续其他工作，主代理在后台收到轮询结束通知后处理结果。

#### 状态映射

| 状态 | 含义 | 后续动作 |
|------|------|----------|
| `Pending` | 等待执行 | 继续轮询 |
| `Running` | 评测进行中 | 继续轮询 |
| `Succeeded` | 评测完成 | 进入任务3 |
| `Failed` | 评测失败 | 展示错误信息，提示重试或排查 |

#### 处理后台返回

后台轮询任务结束后，主代理执行：

1. 读取 `eval-result.json`
2. 根据状态字段执行下方动作

| 状态 | 动作 |
|------|------|
| Succeeded | 进入任务3 |
| Failed | 展示失败原因（从 `eval-result.json` 提取） |

---

### 任务3：展示报告

**用户视角**：看到通过率摘要和在线报告链接。

#### 步骤1：补充 artifacts（按需）

若 `eval-result.json` 中 `artifacts` 字段为空，先后台静默调用 `artifacts` 命令补充。

#### 步骤2：打印摘要

后台静默调用 `summary` 命令（具体命令见 [脚本定义.md](./references/脚本定义.md#7-任务管理)），自动从 `artifacts` 中提取 `report_render_url` 作为在线报告链接。

#### 步骤3：输出格式

**单框架或单模型**：

```
📊 评测结果摘要

运行框架：{runtime-type}
总体通过率：{pass-rate}%（{passed}/{evaluated} 通过）
执行总耗时：{total-duration}

🔗 在线报告：{report_render_url}
```

**多模型横评**：

```
📊 评测结果摘要

运行框架：{runtime-type}

┌──────────┬──────────┐
│ 驱动模型 │ 通过率   │
├──────────┼──────────┤
│ {模型1}  │ {rate}%  │
├──────────┼──────────┤
│ {模型2}  │ {rate}%  │
└──────────┴──────────┘

执行总耗时：{total-duration}

🔗 在线报告：{report_render_url}
```

**运行框架横评**：

```
📊 评测结果摘要

┌──────────────┬──────────┐
│ 运行框架     │ 通过率   │
├──────────────┼──────────┤
│ Claude Code  │ {rate}%  │
├──────────────┼──────────┤
│ OpenClaw     │ {rate}%  │
└──────────────┴──────────┘

执行总耗时：{total-duration}

🔗 在线报告：{report_render_url}
```

**注意事项**：

| 注意 | 说明 |
|------|------|
| 必须展示在线报告链接 | `report_render_url` 提供完整报告，必须输出给用户 |
| 多框架/多模型横评时按框架/模型分组展示 | 让用户能直观对比 |
| 失败用例的具体原因不在摘要中 | 需要时引导用户查看在线报告 |

## Red Flags

| 违规行为 | 正确做法 |
|----------|----------|
| 跳过任务摘要直接提交 | 提交前必须展示完整摘要并等待用户确认 |
| 主代理使用前台轮询阻塞会话 | 默认使用后台轮询（`run_in_background: true`），用户可继续其他工作 |
| 任务成功后未展示 `report_render_url` | 必须输出在线报告链接 |
| 失败时丢弃错误信息 | 必须从 `eval-result.json` 提取错误原因展示给用户 |

**常见借口与纠正**：

| 借口 | 现实 |
|------|------|
| "summary 命令会自动输出链接" | `report_render_url` 来自 `artifacts`，若 `artifacts` 为空需先补充 |
| "用户可以自己去平台查看" | 必须提供直接链接，提升用户体验 |
| "Failed 状态可以悄悄忽略" | 必须告知用户失败原因，便于排查 |

> 通用违规行为见 [SKILL.md Red Flags](./SKILL.md#red-flags)

---

## 常见错误

| 错误 | 原因 | 解决方案 |
|------|------|----------|
| 任务提交失败 | 配置不完整或网络问题 | 检查 `eval-models.json`、`eval-judge.json` 完整性；确认网络连接 |
| 轮询超时 | 评测任务未在 1 小时内完成 | 提示用户稍后用历史任务恢复，或重新轮询 |
| `artifacts` 为空 | 任务成功但报告未生成 | 调用 `artifacts` 命令补充 |
| 在线报告链接打不开 | URL 失效或权限不足 | 提示用户检查登录状态，或在平台首页重新进入 |
